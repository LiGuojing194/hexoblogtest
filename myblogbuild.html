下载git并安装

具体操作自行百度谷歌

将git 与github连接起来

获取ssh公用密钥 ，默认是id_rsa.pub， 我的是保存在C:\Users\用户名\.ssh文件夹里
登录GitHub，找到setting，添加ssh密钥

安装Node.js

自行百度谷歌

安装hexo

安装hexo后，如果bash: hexo: command not found
找到C:\Users\Administrator\AppData\Roaming\npm\node_modules\hexo\bin\，将此目录添加到环境变量中

$ hexo init
$ npm install hexo-deployer-git --save
$ hexo g -d

使用git时，出现的一个让我头疼的问题

晚上还能正常传文件到gitub上，甚至都不用填用用户名和邮箱名。
然而第二天早上起来就一大堆错误，真的很让人头疼。
具体过程略。
解决方案就是
最先： $ git config --global user.name "用户名"
            $ git config --global  user.email "邮箱名"（注意不要落了那些空格）
然后还有一个莫名奇妙的错误：
用$ ssh -v git@github.com后看到一堆错误提示，
就是说在"C:\Windows\System32\config\systemprofile\.ssh"文件夹里找不到公用密钥文件和私用密钥文件。
好生奇怪，不知怎么的密钥读取的默认文件夹突然就变了，于是我只好自己复制密钥到它默认查找的文件夹里，于是问题都解决了
